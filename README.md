# 리드패스트 (ReadFast) — 근거 기반 외국어 읽기 훈련

영어·중국어를 “한국어처럼” 빠르고 정확하게 읽기 위한 설치형 웹앱(PWA).
상업 속독의 사기성 주장(시야폭 확장·RSVP 만능·1만 WPM·포토리딩)을 배제하고,
읽기과학·학습과학에서 **전이가 검증된** 기법만 구현했습니다.

## 핵심 원칙
- **병목은 눈이 아니라 어휘 자동화다.** 눈 운동은 읽기 시간의 ~10%. 지각폭(영어 식별 ~7–8자, 중국어 ~3자)은 고정이라 넓힐 수 없음. 빠른 독자는 시야가 넓어서가 아니라 단어를 빨리 알아봐서 빠르다.
- **속도는 이해로 게이트한다.** 헤드라인 지표는 ERR = 속도 × 이해율. 이해 60% 미만은 0점 → 훑기로 점수를 못 딴다.
- **커버리지 우선 → 속도 → 전략** 순서. 아는 단어 비율 95–98%를 먼저 채운다.
- **회귀·속발음은 보존**(억제하면 이해 손상). **개인 한계는 경험적으로 측정**(고정 WPM 목표 없음).
- 향상은 **학습하지 않은 새 지문(전이)**에서만 인정. 일반 인지능력 향상은 주장하지 않음(근거리 전이만).

## 드릴 (트랙별)
- **커버리지** — 빈도순 어휘 간격반복(SM-2) + 어휘판단 반응시간
- **속도** — **정복(Conquer) 모드**(1차 독해→커버리지 게이트→반복 속독→전이), ERR 정독(이해도 게이트 + 적응형 페이서 staircase), 반복읽기 전이 검사
- **전략** — 모드 선택(훑기/찾기/정독), 논문 3-패스(Keshav), 자기설명·인출(지연)
- **중국어 전용** — 단어 경계 분할, 글자 인지속도·의미 부수
- **시야 활용** — 주제문 포레이징(폭 확장 아님)
- **원리 탭** — 위 전부의 메커니즘·증거 강도·미신 반박·**과부하→회복→강화** 프레이밍·출처

## 과부하 → 회복 (이 사용자 맞춤)
한계 위 난이도로 과부하 → 회복으로 강화하는 방식을 따르되, **부하 축을 바로잡음**: 어휘·밀도·길이·인출에 과부하를 걸고 **눈 속도엔 걸지 않음**(이해 붕괴=훑기). 쉬운 지문(영 C2 이하/중 HSK6 이하)은 기본 숨김 — 기본 난이도가 최상위. 회복(간격·deload·수면)은 새 단어 학습을 공고화하지 속도를 빠르게 하지 않음(정직 고지).

## 구조
```
index.html            앱 셸
css/styles.css         디자인 시스템(라이트/다크, 반응형)
js/app.js              라우터 + 홈/훈련/내 글/기록 뷰
js/store.js            localStorage 상태·SR 스케줄러·적응형 staircase
js/content.js          코퍼스 로딩·토큰화·자동 클로즈
js/theory.js           「원리」 탭
js/drills/*.js         드릴 10종 + 공용 UI(shared.js) + 레지스트리(index.js)
data/*.json            지문 51(EN 1–6 C2+ 포함, ZH 1–6 HSK6+ 포함)·어휘(EN 202/ZH 174)·분할 31
sw.js, manifest        PWA(오프라인·설치)
_build/                데이터 생성 스크립트(배포 제외)
```

## 콘텐츠
모든 지문은 저작권 안전을 위해 **100% 원본 생성**, 이해 문항은 “지문만 보고 푸는” 적대적 검증을 거쳤습니다.
사용자가 「내 글」에 붙여넣은 책·논문 단락의 이해 문항은 자동 클로즈(검증 안 됨, 자기 점검용)입니다.

## 실행 / 배포
- 로컬: 정적 서버로 폴더를 서빙(`python -m http.server` 등) 후 접속. (ES 모듈·fetch 때문에 file:// 직접 열기는 제한)
- 배포: GitHub Pages(`.nojekyll` 포함) 또는 Netlify에 폴더 그대로 업로드. HTTPS에서 PWA 설치 가능.

## 면책
교육·자기훈련용. 학습 효과는 개인차가 있으며, 이 앱은 과장된 속독 효과를 약속하지 않습니다.
